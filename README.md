PDGo
Overview

PDGo is an internal employee directory and self-service application developed for the Santa Barbara Public Defender’s Office as a senior capstone project. The goal of the project was to streamline internal workflows by centralizing employee information, requests, and training records into a single, easy-to-use platform.

The application was designed for internal use only and focuses on improving efficiency, accessibility, and organization for staff members and supervisors.

Features

Employee Directory – Centralized employee information with role and department visibility

Self-Service Requests – Submission and tracking of internal requests such as PTO and general service requests

Training & Certifications – Tracking employee participation in required or optional training programs

Role-Based Access – Permissions managed through Azure Active Directory

Web-Based Interface – Accessible through Microsoft Power Apps

Tech Stack

Microsoft Power Apps – Front-end application development

SharePoint – Data storage and list management

Azure Active Directory – Authentication and role-based access control

Microsoft 365 – Platform integration and deployment

Project Context

This project was completed as part of a Senior Capstone course at California State University, Los Angeles. The project was sponsored by the Santa Barbara Public Defender’s Office and developed using an Agile/Scrum methodology.

I served as the team lead, coordinating task assignments, sprint planning, deadlines, and client communication while also contributing to application design and development.
